torch>=2.1
# The old `<4.45` upper bound guarded against transformers 4.46+ removing
# `forced_bos_token_id` from `PretrainedConfig`, which Florence-2's
# `trust_remote_code` modeling file still reads. That guard is now redundant:
# `caption.py` patches Florence2Config at load time (see the compat shim
# around its `_get_dyn_cls` call), which is the actual fix.
#
# The bound was also unsatisfiable on Python 3.13 -- despite this file's
# stated 3.10-3.13 target. `transformers<4.45` pins `tokenizers<0.20`, whose
# last release predates cp313, so pip fell back to a Rust source build that
# dies on `PyO3 0.21 max supported version 3.12`. Building it against the
# stable ABI does not help either: tokenizers 0.19 calls the non-abi3
# `PyUnicode_FromKindAndData`.
#
# Keep the `<5` cap. transformers 5.x is a major API break and this codebase
# (caption.py's Florence-2 loader, layout.py's LayoutLMv3 branch) is written
# against the 4.x surface. 4.46+ already pulls a tokenizers with cp313 wheels,
# which is all Python 3.13 needed.
transformers>=4.46,<5
